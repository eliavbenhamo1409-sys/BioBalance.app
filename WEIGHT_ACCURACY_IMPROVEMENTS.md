# 🎯 שיפורי דיוק המשקל - BioBalance

## 📊 הבעיה המקורית
סטייה של ±50 גרם בהערכת משקל (עד 25% שגיאה במנה של 200g)

## ✅ 5 שיפורים שיושמו

### 1️⃣ **טבלת משקלים סטנדרטית** (`referenceWeights.js`)

#### מה זה?
מאגר משקלי מנות אמיתיים מ-USDA ומשרד הבריאות.

#### איך זה עוזר?
```javascript
// לפני
AI: "אורז מבושל = 130g"  ← לא עקבי

// אחרי
AI: "אורז מבושל = 130g"
System: "⚠️ מנה רגילה של אורז = 150-180g"
→ מתקן ל-165g (ממוצע משוקלל)
```

#### דוגמאות מהמאגר:
```javascript
STANDARD_PORTIONS = {
  rice_cooked: {
    portions: [
      { description: 'מנה קטנה', grams: 100 },
      { description: 'מנה רגילה', grams: 150 },
      { description: 'מנה גדולה', grams: 250 },
    ],
    density: 0.65  // g/ml לחישוב נפח
  },
  chicken_breast: {
    portions: [
      { description: 'חתיכה בינונית', grams: 150 },
      { description: 'חזה שלם', grams: 250 },
    ],
    density: 1.05
  }
}
```

### 2️⃣ **אובייקטי ייחוס מדויקים**

#### מה זה?
מידע על גדלים אמיתיים של כלים וציוד.

#### השפעה:
```
לפני:
👁️ AI: "נראה כמו מנה בינונית"

אחרי:
📏 AI: "רואה מזלג (19 ס\"מ) לצד המנה"
      → "חתיכת העוף פי 1.4 ממזלג = ~27 ס\"מ"
      → "עוף בעובי 2 ס\"מ"
      → "נפח משוער: 27 × 10 × 2 = 540 ml"
      → "צפיפות עוף: 1.05 g/ml"
      → "משקל: 540 × 1.05 = ~567g"  ❌ חריג!
      → תיקון: חזה עוף סטנדרטי 150-180g ✓
```

#### אובייקטי ייחוס זמינים:
- 🍴 מזלג: 19 ס"מ
- 🥄 כף: 15 ס"מ
- 🍽️ צלחת רגילה: 26 ס"מ קוטר
- 🥗 צלחת סלט: 20 ס"מ קוטר
- 🥣 קערת מרק: 15 ס"מ קוטר
- 🥛 כוס: 7 ס"מ קוטר, 12 ס"מ גובה

### 3️⃣ **Stage A.5: Weight Calibration**

#### מה זה?
שלב חדש שמשווה את ההערכה של ה-AI למנות סטנדרטיות.

#### לוגיקה:
```javascript
function stageA5_WeightCalibration(item) {
  const aiEstimate = item.estimated_grams;        // 130g
  const standard = findClosestPortion(item.name); // 150g
  
  const diff = Math.abs(standard - aiEstimate);   // 20g
  const diffPercent = (diff / aiEstimate) * 100;  // 15.4%
  
  if (diffPercent > 20) {
    // שילוב משוקלל: 70% סטנדרט + 30% AI
    return standard * 0.7 + aiEstimate * 0.3;     // 144g
  }
  
  return aiEstimate; // סטייה קטנה - לא מתקן
}
```

#### תוצאה:
```
לפני Calibration:
• אורז: 130g
• עוף: 200g

אחרי Calibration:
• אורז: 144g ✓ (מנה רגילה תקנית)
• עוף: 165g ✓ (התאמה למנה בינונית)
```

### 4️⃣ **פרומפט משופר עם הנחיות גיאומטריות**

#### מה השתנה?
הפרומפט ל-AI עכשיו כולל:

1. **שיטת עבודה ב-3 שלבים:**
   ```
   שלב 1: זהה אובייקטי ייחוס
   שלב 2: זהה מזון
   שלב 3: הערך משקל (עם גיאומטריה!)
   ```

2. **חישובים גיאומטריים:**
   ```
   • אורז בצורת כיפה → נפח חצי כדור
   • חזה עוף שטוח → נפח מנסרה
   • סלט → נפח לא סדיר
   ```

3. **בדיקת צפיפות:**
   ```
   סלט:  0.4 g/ml (קליל)
   אורז:  0.65 g/ml
   בשר:   1.05 g/ml (כבד)
   ```

4. **בדיקת סבירות:**
   ```
   אורז מבושל: מנה רגילה 150-200g
   אם ההערכה שלך 90g או 350g → בדוק שוב!
   ```

#### לפני ואחרי:
```
❌ לפני:
"זהה מזון והערך משקל"
→ AI: "נראה כמו 180g" (ללא הסבר)

✅ אחרי:
"זהה אובייקט ייחוס → חשב יחס → השתמש בצפיפות → השווה לסטנדרט"
→ AI: "מזלג 19 ס\"מ, עוף פי 1.3 = 24.7 ס\"מ, עובי 2 ס\"מ, רוחב 8 ס\"מ
      → נפח: 24.7 × 8 × 2 = 395 ml
      → צפיפות עוף: 1.05 g/ml
      → משקל גולמי: 395 × 1.05 = 415g
      → חריג! מנה סטנדרטית = 150-180g
      → הערכה מתוקנת: 165g"
```

### 5️⃣ **Image Detail: High**

#### מה זה?
שדרוג פרמטר ה-API של OpenAI.

```javascript
// לפני
image_url: {
  detail: 'low'  // רזולוציה נמוכה, מהיר אבל פחות מדויק

// אחרי

```

#### השפעה:
- רזולוציה גבוהה יותר → זיהוי טוב יותר של אובייקטי ייחוס
- פירוט טוב יותר של מרקם וטקסטורה
- הערכת נפח מדויקת יותר

## 🎓 מה ה-AI לומד?

הפרומפט החדש מלמד את ה-AI:

1. **לחפש אובייקטי ייחוס באופן אקטיבי**
   - "סרוק את התמונות וזהה מזלג/כף/צלחת"
   
2. **לחשוב בגיאומטריה**
   - "מה הצורה? כיפה? מלבן? נפח?"
   
3. **להשתמש בצפיפות**
   - "סלט קליל, בשר כבד"
   
4. **לבדוק סבירות**
   - "האם זה הגיוני? מה המנה הסטנדרטית?"

5. **להסביר החלטות**
   - שדה `reasoning` מכריח את ה-AI לנמק
